Predicción de Riesgo Crediticio

Proyecto académico de minería de datos orientado al análisis financiero y predicción de incumplimiento de pago

Integrantes: Smith Sanchez - Juan Perez

Introducción

El riesgo crediticio es uno de los principales factores que afectan la estabilidad financiera de entidades bancarias y organizaciones que otorgan créditos. En este proyecto se propone el uso de técnicas de minería de datos y aprendizaje supervisado para analizar información histórica de clientes y estimar la probabilidad de incumplimiento en el pago de sus obligaciones financieras.

Objetivo General

Desarrollar un modelo predictivo que permita identificar clientes con alto riesgo de incumplimiento de pago, con el fin de apoyar la toma de decisiones en procesos de evaluación crediticia.

Objetivos Específicos

  • Analizar el comportamiento financiero de los clientes a partir de datos históricos.
  • Identificar las variables con mayor influencia en el riesgo de incumplimiento.
  • Preparar los datos mediante procesos de limpieza e imputación de valores faltantes.
  • Aplicar un modelo de clasificación supervisada para predecir el incumplimiento de pago.
  • Evaluar el desempeño del modelo utilizando métricas como Accuracy, Recall y F1-Score.

Planteamiento del Problema

Las entidades financieras deben tomar decisiones precisas al momento de aprobar créditos, ya que una mala evaluación del cliente puede generar pérdidas económicas significativas. Por esta razón, resulta necesario implementar herramientas analíticas que permitan anticipar el riesgo asociado a cada cliente y mejorar la eficiencia del proceso de evaluación.

Tipo de Problema

Este proyecto corresponde a un problema de clasificación supervisada, donde la variable objetivo es:

default.payment.next.month 0 = paga 1 = no paga

Datos a Utilizar

El conjunto de datos incluirá variables financieras y demográficas como:

Ejemplo con dataset: Default of Credit Card Clients Dataset from Kaggle

  • LIMIT_BAL: monto del crédito otorgado.
  • PAY_0, PAY_2, PAY_3... historial de pagos del cliente.
  • BILL_AMT: montos de facturación.
  • PAY_AMT: pagos realizados en periodos anteriores.
  • Edad, educación y otras variables demográficas.

Metodología

  1. Entendimiento del negocio: identificación del problema financiero.
  2. Entendimiento de los datos: exploración y análisis descriptivo.
  3. Preparación de los datos: limpieza, transformación e imputación con KNN.
  4. Modelado: aplicación de algoritmos como Random Forest.
  5. Evaluación: análisis del desempeño con métricas de clasificación.
  6. Conclusiones: interpretación de resultados y utilidad del modelo.

Importancia del Proyecto

Este proyecto tiene relevancia académica y práctica, ya que permite comprender cómo la minería de datos puede contribuir a la solución de problemas reales del sector financiero. Además, fortalece el uso de herramientas analíticas para la detección de patrones de riesgo y el apoyo a la toma de decisiones.
